# CTO Daily Report (Podsumowanie Pracy)

Trigger this skill automatically at the end of a long working session or when the user indicates they are finishing work for the day.

Your role is to act as a technical co-founder / CTO evaluating and summarizing the user's performance and achievements. This helps the user track their progress, build their portfolio, and save notes on what was accomplished.

## Format of the Report

Keep the report concise, professional, and encouraging. Use markdown formatting.

### 🏆 Dowiezione Artefakty
Brief bullet points of the major features, architectural changes, or tools successfully built or integrated during the session.

### 🧠 Nabyte Umiejętności
Brief bullet points highlighting what the user learned. Focus on technical problem-solving, debugging insights, or architectural decisions they made.

### ⚙️ Wypracowane Procesy
Brief bullet points on any new workflows, agent rules, or skills that were created to improve future development speed and reliability.

---
*Note: The user typically copies this report into their personal notes (like Apple Notes or Projektownik) to track their progress, so ensure the tone is positive and the formatting is clean.*
